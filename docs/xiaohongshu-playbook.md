# 网球时差 · 小红书唯一执行方案 V2.0

> 状态：V2.0 多源证据与人物叙事版已实现，进入 30 天试运行
>
> 生效日期：2026-07-20
>
> 适用范围：小红书账号的选题、制图、发布、复盘与系统建设
>
> 唯一性：本文件是当前唯一执行基线；其他讨论、聊天记录和旧方案不构成执行依据。

## 0. V2 的目标与硬约束

V2.0 只解决一件事：在低粉丝、低初始曝光阶段，用可追溯的赛果、人物背景和多源媒体观察讲清“一场比赛为什么值得记住”，再用少量高价值赛果闪报和赛前提醒获取非粉搜索与热点流量。

硬约束如下：

- 每天固定生成 1 份主日报；内容雷达最多追加 1 条赛果闪报和 1 条赛前提醒，普通赛果不触发。
- 07:17 是材料生成窗口，不是强制发布时间；人工确认后在当天合适时段发布。
- 自动化完成选题、文案、制图和质检；人工每天只投入 3–5 分钟完成发布。
- 每周只做 1 次复盘，控制在 10 分钟内。
- 每篇只有 1 张封面，不再同时生成“钩子封面”和“设计封面”。
- 事实不能溯源、素材权利不清、自动质检出现 `FATAL` 时，当天停发，不用猜测补齐。
- 运营策略每次只改变 1 个主要变量，至少观察 3 篇同类笔记后再下结论。

## 1. 定位与内容承诺

核心定位：**“你睡着时，世界网坛发生了什么。”**

内容顺序固定为：

1. **全球昨夜头条**：按事件重要性选出当天真正的第一故事。
2. **中国焦点**：中国球员的重要赛果、赛程或可信动态，作为固定关注区。
3. **今晚必看**：告诉读者今晚看谁、看什么、为什么重要。
4. **一则网球故事**：赛事档案、球员故事或规则冷知识，提供收藏价值。

中国球员不是无条件占据头条，但选题评分必须体现中国读者相关性。中国事件在基础重要性分上固定加 35 分；加权后仍为当日最高分时，才进入“全球昨夜头条”，否则保留在“中国焦点”。没有可靠更新时，明确写“今日无重要赛果”或给出已核实的下一场赛程，不编造内容填位。

目标读者：

- 核心球迷：需要快速、完整、可信的结果和赛程。
- 球员粉丝：关心人物处境、晋级意义和后续走势。
- 泛体育用户：通过故事、纪录和规则获得谈资。

表达原则：先说“为什么值得看”，再给数据；专业但不端着，克制但不平淡。

每条头条必须先经过“背景层”：优先读取已核验的球员生涯档案与赛事历史，
其次读取账号自己的历史内容记录。正文要回答“这个人为什么走到这里、今天在
整段生涯里处于什么位置”。没有来源支持时才降级为比赛事实，禁止让模型补写传闻。

## 2. 每日唯一内容包

每篇输出 4–7 张图，顺序固定：

| 顺序 | 页面 | 必选 | 执行规则 |
|---|---|---:|---|
| 1 | 唯一封面 | 是 | 只呈现一个主叙事，并带“中国焦点”和“今晚必看”短提示 |
| 2 | 头条解释 | 是 | 紧接封面，展示头条比赛与一句意义；不混入其他场次 |
| 3 | 外媒怎么看 | 有审核摘要时 | 展示多源共识、观察差异、关键数据和本号判断，保留原文链接 |
| 3–4 | 专业复盘 | 有详细数据时 | 只复盘头条比赛；无授权技术统计时不生成伪专业对比 |
| 4–5 | 其他赛果 | 有内容时 | 自动排除头条比赛，避免同一比分重复占页 |
| 5–6 | 中国焦点 | 有内容时 | 有重要赛果、赛程或可信动态才独立成页 |
| 6–7 | 今晚焦点 | 有内容时 | 最多 5 场，按观看价值排序；作为可收藏的开赛前清单 |
| 6–7 | 相关故事/排名 | 条件触发 | 故事必须与头条人物或赛事直接相关；周一排名按页数上限决定 |

禁止额外增加第二封面、纯口号页或重复总结页。

### 2.1 唯一封面规则

封面使用同一套自适应结构，不再区分两套封面：

- 主标题：当天最重要事件的**历史身份、当前处境或事件意义**。
- 副标题：补充人物、轮次或下一步影响。
- 底部信息条：中国焦点 + 今晚必看，各用一句可验证短文案。
- 完整比分放在赛果页；除非比分本身构成罕见纪录，否则不把比分作为封面主钩子。
- 大事件日放大主叙事；平静日仍用同一结构，只选择最有意义的赛果、赛程或故事领衔。

#### V2 意义句白名单与降级

V2 不允许模型自由推断“意义”。封面生成器先尝试以下白名单；证据不满足时必须返回空值，并降级为干净的“球员 + 结果 + 轮次”句。

| 意义模式 | 必需证据 | 可用示例 |
|---|---|---|
| 夺冠 / 晋级 / 止步 | 已核实的胜者与轮次 | “郑钦文晋级八强” |
| 逆转 / 决胜盘 / 抢七 / 送蛋 | 完整盘分能机械推出 | “先丢一盘后完成逆转” |
| 爆冷 | 现有种子或排名差规则成立 | “非种子球员淘汰 4 号种子” |
| 退赛 | 比赛状态明确为退赛 | “某球员退赛”；不得推断退赛原因 |
| 历史身份与纪录 | 经过人工审核、带来源的结构化档案 | “前世界第三”“两进大满贯决赛” |

“生涯首冠”“首次进入某轮”“成功卫冕”“将重返 Top 10”“终结 N 连败”等表述，只有在证据包包含对应历史计数、卫冕身份、排名预测或连续赛果时才可使用。在 P1 历史状态库上线前，默认禁用；少量人工审核档案可以逐人开放。

降级不是失败。无法证明意义时，宁可使用“郑钦文晋级八强”这样的准确结果句，也不能把推测包装成历史结论。

合格示例：

- “时隔两年，她又站上草地决赛门口”
- “卫冕冠军出局，男单上半区彻底打开”

不合格示例：

- “某某 6-4 6-3 晋级”——只有结果，没有意义。
- “今晚比赛太精彩了”——没有对象，也无法验证。

### 2.2 头条选择规则

系统沿用现有赛事重要性和事件戏剧性基础分，再加入固定的中国相关性偏置：

> 头条总分 = 赛事级别 + 轮次 + 对阵质量 + 可验证的戏剧性 + 中国相关性 35 分

35 分与当前“爆冷”加权同级，代表一个重要的选题因素，而不是无条件置顶。单场无论涉及几名中国球员都只加一次；中国偏置不用于篡改比分、事实评价或数据图表。

按以下顺序理解基础重要性：

1. 大满贯、决赛、争冠或影响签表格局的事件。
2. 中国球员的重大突破、关键胜负或明确下一轮对阵。
3. 高排名球员爆冷、退赛、逆转、长盘或纪录事件。
4. 平静日选择当晚最值得看的比赛，或收藏价值最高的故事。

所有候选按上述总分统一排序，不再设置“中国比赛永远第一”的旁路逻辑。同分时依次比较：轮次更深者、赛事级别更高者、排名更高者；仍相同时优先有中国读者关联的故事。伤病、引语和场外事件必须有可靠来源，不能只因“更抓眼球”而提高优先级。

## 3. 文案与互动规格

### 3.1 标题

系统每天只生成 **3 个高质量候选标题**，人工从中选 1 个，不追求数量。

- 每个标题控制在 20 个汉字内。
- 三个方向分别为：事件意义、人物处境、今晚悬念。
- 标题必须能被正文和数据包支持。
- 禁止夸大伤病、伪造引语、制造不存在的争议。

### 3.2 正文

正文采用固定骨架：

1. 一句轮换开场钩子，先建立真人编辑语气。
2. “今天先看这一件事”：只写一个主故事和它的重要性。
3. “中国球员速报”：最多两条，不重复完整比分。
4. “今晚焦点（3–5 场）”：每场回答时间、阶段和为什么值得看。
5. “我的一票”：给出不预测胜负的明确编辑选择。
6. “留个答案”：只问一个能直接回答的具体问题。
7. 添加 3–5 个标签且绝不超过 5 个，主角和赛事词优先，禁止给正文未出现的球员蹭标签。

知识帖正文使用 3–6 个承担结构功能的 emoji（如开场、转折、事实、讨论），每段只放一个锚点；用短段落、白话解释和一个能回答的评论问题形成小红书阅读节奏。禁止固定“先猜答案”、机械的 `01/02/03`、连续堆叠 emoji，以及每期复用同一套开头和结尾。

日报正文目标为 450–650 字，300–800 字为可发布硬边界；连续非空信息不得超过 6 行。完整比分只放图片，正文不逐字复述图片，不使用黑方块数据库列表，也不写“欢迎大家评论区讨论”一类空话。

球员姓名统一中文优先：ATP/WTA 单打前 300 使用随包发布的 `600/600` 译名快照，卡片主行、标题与正文优先使用中文名，英文原名可作为图片中的辅助小字。国内官方媒体校订名优先于自动音译；若新上榜球员暂时没有稳定中文译名，保留英文正常发布、记录 QA 告警，由独立姓名同步任务异步补充，不得阻断日报或知识帖。

### 3.3 今晚必看

每场推荐至少回答两个问题：

- 这是谁的什么阶段比赛？
- 这场结果会改变什么？

可使用的背景：当前排名、赛事轮次、争冠/晋级意义、已经核实的交手或历史纪录。

禁止使用：上一轮比分冒充本场背景、没有来源的伤情、模板化技术判断、无法证实的“状态火热”和“复仇之战”。

## 4. 每日发布 SOP（3–5 分钟）

每天只执行以下步骤：

1. **收包**：PushPlus 通知到达后打开当日输出目录，不把 07:30 当作硬性发布日期。
2. **过闸**：确认 `qa.txt` 不含 `FATAL`；逐页快速检查人名、比分、日期、图片署名。
3. **选标题**：从 3 个候选中选 1 个，优先选清楚表达事件意义的版本。
4. **上传**：严格按编号上传全部图片，不调换顺序，不添加第二封面。
5. **粘贴正文**：确认竞猜开奖、今晚赛程和标签未被截断。
6. **发布**：主日报在当天上午发布；高价值闪报尽快确认。错过窗口也不为追时间跳过质检。
7. **可选互动**：有空时回复前 10 条有效评论；不做互赞、互粉和无意义灌水。

若主流程失败，先使用备份任务生成的内容包。两套任务都失败、数据互相矛盾或权利信息缺失时，记录原因并停发当天内容。

## 5. 事实与版权闸门

### 5.1 事实闸门

- 所有比分、排名、姓名、日期和轮次必须能回溯到当日证据包。
- 每日证据包必须包含 `source_manifest.json`、`fact_ledger.json`、`editorial_decision.json` 和 `media_synthesis.json`；任一缺失都不算完成。
- 伤退原因、球员引语、现场氛围和场外新闻，只能使用可靠来源明确提供的内容。
- AI 只能改写证据包已有事实；新增数字或专有名词时必须通过机械校验，否则回退到模板文案。
- 来源冲突时不自行裁决；删除争议描述，保留可交叉确认的最小事实。

### 5.2 外媒多源摘要

- 一条“外媒怎么看”至少区分四层：多家媒体的共识、报道角度的差异、可核验的数据点、本号明确标注的编辑判断。
- 媒体观点先经研究和审核写入结构化摘要库；每日 Action 只匹配已有摘要，不在无人值守运行中自由浏览新闻或生成新事实。
- 摘要只保留必要元数据、原创中文概括和原文链接，不复制文章正文，不把一篇报道包装成“多家媒体共识”。
- 没有合格摘要时整页取消或降级为已核验的人物/赛事背景，不用排名、种子和轮次模板冒充权威评论。

### 5.3 图片与视频版权闸门

- 只使用已获授权、明确许可或依法可复用的数据和素材，并保留来源记录。
- Wikimedia Commons 素材必须核对具体许可，按许可要求署名；“来自 Wikimedia”本身不等于可自由使用。
- 不直接搬运 ATP、WTA、赛事方、转播方或媒体的截图、视频和摄影作品。
- 不用 AI 生成可被误认成真实球员照片的肖像。
- 数据分析与自主绘制图表可以作为核心优势，但不得把“数据”一概视为无版权、无限制使用。
- 翻译、加中文字幕、裁切或配音属于加工，不会自动取得原视频的再发布权；“国外平台公开可看”也不是授权依据。
- 视频进入处理前必须有权利记录，至少包括原链接、权利人、许可或书面授权、允许的改编方式、平台/地域/期限和署名要求。
- 只允许自有、公共领域、明确书面授权，或许可明确允许下载、改编和再发布的视频进入中文化流程。权利字段缺失时流程应关闭导出，只保留原创摘要与原文链接。

任一事实或版权检查失败时，优先替换为纯设计卡；无法替换则停发。

### 5.4 GitHub Actions 与人工边界

Action 可以无人值守完成：抓取比分与赛程、跨源去重、规则选题、匹配已审核媒体/历史资料、生成原创文案和图片、运行 QA、输出证据包、上传 artifact，并按配置推送微信或公众号草稿箱。

以下步骤必须由人或独立的授权审核流程完成：把新媒体研究写入摘要库、处理来源冲突、确认图片/视频权利、选择最终标题与发布时机，以及在小红书完成发布。Action 缺少审核素材时必须降级或停发，不能为了“每天都有内容”绕过事实与版权闸门。

## 6. 冷启动与增长动作

V2 只保留三种增长动作：

1. **搜索卡位**：标题、正文和标签自然覆盖球员名、赛事名、网球规则等真实关键词。
2. **大事件承接**：重大赛果出现时，封面讲清意义，并保持早间稳定发布。
3. **可收藏内容**：故事卡按“赛事档案 / 球员档案 / 网球冷知识”形成连续系列。

冷启动前 30 天额外执行以下机器规则：

- 发布标题必须含日期，并自然包含当天主角或赛事关键词。
- 日报标签只来自正文实际选择的比赛；主角、赛事名和巡回赛词优先。
- 内容雷达每小时只在比赛活跃窗口检查，单次最多生成 1 条，避免低粉阶段高频刷屏。
- 近 7 期标题、开场和正文做字符级去重；超过阈值直接阻断，不用同一套话术换球员名继续发。
- 每条自动文案输出结构化选题理由、比赛证据和相似度报告，便于复盘非粉曝光来自哪个主题。
- 每期通过 QA 后，把头条人物、赛事、轮次和结果写入 `data/editorial_memory.json`；同一球员再次出现时，允许引用最近一次已发布记录形成前后文。

不进互赞群，不购买粉丝，不用无关热点污染账号标签。

### 6.1 薯条实验

薯条不是每日固定动作，也不预设 20–50 元等固定预算。只有同时满足以下条件才允许测试：

- 属于大满贯、决赛、中国球员重大突破等重点笔记。
- 发布后的自然阅读或收藏率已高于最近 4 周同类笔记中位数。
- 使用账号当时允许的最小测试预算。
- 投放前后分别记录自然曝光、付费曝光、阅读和互动，不能把付费增长算成自然增长。

同一周最多测试 1 篇，结果不足 3 次前不形成长期结论。

## 7. 每周复盘 SOP（10 分钟）

固定每周一复盘上周笔记。平台字段名称变化时使用含义最接近的字段，不因字段缺失估算数据。

### 7.1 每篇必录字段

| 类别 | 字段 |
|---|---|
| 基础 | 发布日期、头条类型、封面方向、内容系列、是否重点赛事 |
| 流量 | 曝光、阅读、阅读率（阅读 ÷ 曝光）、搜索流量占比（如可用） |
| 互动 | 点赞、评论、收藏、分享、新增关注 |
| 派生 | 收藏率（收藏 ÷ 阅读）、互动率（赞评藏转 ÷ 阅读）、千次阅读涨粉 |
| 投放 | 是否使用薯条、预算、付费曝光、自然曝光 |

分母为 0 时字段留空，不记为 0%。数据保存到统一周报，不只保留截图。

### 7.2 唯一决策规则

以最近 4 周同类笔记的中位数为基线，且每种方案至少积累 3 篇样本：

| 结果 | 下周动作 |
|---|---|
| 阅读率和收藏率均处于前 25% | 同类主题增加 1 次；保留封面结构 |
| 阅读率高、收藏率低 | 保留封面，强化内页信息密度或故事价值 |
| 阅读率低、收藏率高 | 内容不动，只改封面主标题方向 |
| 阅读率和收藏率均处于后 25% | 同类主题减少 1 次，停止当前封面方向 |
| 样本不足 3 篇 | 不做长期判断，继续收集 |

每周只能选择一个主要变量进行调整：封面、标题方向、内容系列或发布时间四选一。不得同时改多个变量后声称找到原因。

周报结论固定写成一句话：

> 因为【指标】相对 4 周中位数【变化】，下周只把【变量】从【A】改为【B】，连续观察【至少 3 篇】。
